# Frontend Engineering Skill

This skill provides a set of directives for enhancing the frontend experience of the application.

## Core Directives

### 1. Visual Excellence
- **Aesthetic**: Aim for a premium, modern, and high-contrast look.
- **Theme**: Use a dark purple and black mix for the background with vibrant, "popping" highlights.
- **Typography**: Prioritize modern, clean fonts (e.g., Inter, JetBrains Mono) with high legibility.
- **Interactivity**: Use smooth transitions, subtle micro-animations, and clear feedback for user actions.
- **Popping Elements**: Buttons, status indicators, and important text should have strong visual presence (shadows, glows, or gradients).

### 2. Implementation Rules
- **Backend Integrity**: Frontend changes MUST NEVER break or negatively impact the backend API or data flow.
- **Minimal Impact**: Avoid refactoring large portions of the codebase unless necessary for styling improvements.
- **Tailwind CSS**: Leverage Tailwind's utility classes for styling to maintain consistency and ease of maintenance.
- **Responsiveness**: Ensure the UI remains functional and looks great on all screen sizes.

### 3. Workflow
1.  **Analyze**: Understand the existing UI structure and component hierarchy.
2.  **Design**: Plan the visual enhancements, focusing on the dark purple/black theme.
3.  **Implement**: Apply styling changes carefully, verifying each step.
4.  **Verify**: Test the UI changes locally and ensure backend functionality is preserved.
